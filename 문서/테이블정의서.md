## 1. OpenSearch 인덱스 정의

### 1-1. my_vector_index (벡터 검색 인덱스)

> RAG 파이프라인의 핵심 인덱스. 문서를 청크 단위로 분할하여 임베딩 벡터와 함께 저장

| 필드명 | 타입 | 설명 | 비고 |
|--------|------|------|------|
| id | integer | 원본 문서 ID | 원본 데이터의 id |
| title | text | 문서 제목 | 검색 및 참조 출처 표시용 |
| category | keyword | 문서 카테고리 | 정확 매칭 검색용 |
| chunk_text | text | 분할된 텍스트 청크 | 청크 크기: 600, 오버랩: 100 |
| embedding | knn_vector | 텍스트 임베딩 벡터 | 차원: 1024, 모델: bge-m3 |

**인덱스 설정**

| 항목 | 값 |
|------|----|
| knn | true |
| 임베딩 모델 | bge-m3 (Ollama) |
| 벡터 차원 | 1024 |
| 검색 방식 | KNN (K-Nearest Neighbor) |
| 검색 결과 수 (K) | 4 |

## 3. API 데이터 구조 (Spring ↔ FastAPI)

### 3-1. 요청 데이터 (ChatRequest)

> Spring → FastAPI /predict 전달 구조

| 필드명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| question | string | 사용자 질문 | 카메라 성능이 좋은 스마트폰은? |

### 3-2. 응답 데이터 (ChatResponse)

> FastAPI → Spring 반환 구조

| 필드명 | 타입 | 설명 | 비고 |
|--------|------|------|------|
| prediction | string | AI 답변 텍스트 | LLM 생성 답변 |
| references | array | 참조 문서 목록 | 최대 K개 (기본 4개) |

### 3-3. 참조 문서 항목 (ReferenceDto)

| 필드명 | 타입 | 설명 | 비고 |
|--------|------|------|------|
| title | string | 참조 문서 제목 | OpenSearch 검색 결과 |
| category | string | 참조 문서 카테고리 | OpenSearch 검색 결과 |
| content | string | 참조 문서 내용 일부 | 최대 200자 |
